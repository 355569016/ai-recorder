# AI录音卡 App 页面效果设计需求文档 v3.0

版本：v3.0  
日期：2026-07-07  
Andy.Ma

本文用于产品评审、UI 细化、前端拆解、后端接口梳理、硬件/固件联调。本文不新增 PRD 范围外功能；未确定项统一标注【待确认】。

## 1. 设计目标

| 目标 | 说明 |
|---|---|
| 可评审 | 通过浏览器直接查看 App 页面实际效果、主流程和状态 |
| 可开发 | 每个页面明确字段、状态、交互、依赖和验收点 |
| 可联调 | 明确哪些数据来自 BLE、设备协议、本地库、云端账号、云端文件、转写服务、AI 服务、用量服务 |
| 可迭代 | 原型文件独立，不覆盖旧版 PRD 和交互式 PRD |

## 2. 视觉与交互原则

| 项 | 设计要求 |
|---|---|
| 视觉基调 | 白底、黑色主按钮、浅灰信息区、细边界、少量状态色 |
| 页面密度 | 接近工具型 App，避免营销式大图和复杂装饰 |
| 信息层级 | 顶部导航 -> 紧凑设备/同步状态 -> 首页文件列表/表单 -> 主操作 |
| 圆角 | 卡片 16px，按钮 8-10px，状态标签胶囊形 |
| 录音主按钮 | 固定底部中间，视觉优先级高于普通 Tab |
| 底部弹层 | 用于录音方式、生成方式和用量确认，半屏弹层结构 |
| 状态色 | 绿色成功、蓝色设备/来源、琥珀处理中/额度、红色异常、紫色 AI |
| 文案 | 简洁直接，异常必须说明原因、影响和下一步 |

## 6. 前端实现注意事项

| 类别 | 要求 |
|---|---|
| 页面结构 | 底部导航固定，录音主按钮居中浮起，不纳入普通 Tab；文件列表并入文件首页，不单独拆页 |
| 本地状态 | 需要维护登录态、绑定态、连接态、录音态、同步队列、文件列表、任务状态、用量账本 |
| 数据来源 | 本地数据库是文件列表主来源；BLE 只负责设备状态、录音命令、同步传输 |
| 文件来源 | `file_source` 必填，枚举为 `hardware_recording` / `imported_audio` |
| 原型链路 | 仅用于连接不同状态页、非用户可见的跳转按钮必须放在手机框外的原型链路区，不放入 App 页面 |
| 弹层 | 录音方式和生成方式统一使用半屏弹层，支持点击遮罩关闭 |
| 异常 | 每个功能入口前置检查失败时跳转对应异常状态或引导页 |
| 可访问性 | 主按钮、危险操作、开关、列表项需要明确可点击区域和状态 |
